# 물류창고_출고영역_구현_내역

## 1. 구현 개요

출고 관리를 `매장 출고` 중심 구조에서 `매장 출고 + 창고간 이동 + 순환재고 판매 출고`를 함께 조회하는 통합 구조로 확장했다.

핵심 반영:

- 출고 단일 읽기 모델(`warehouseOutbound`)로 3유형 통합
- 리스트에 유형 필터 추가
- 상세 액션을 유형별로 분기(`STORE_OUTBOUND`만 처리 가능)

## 2. 메뉴 및 라우트

- `/warehouse/outbound`
- `/warehouse/outbound/:id`

## 3. 스토어 구현

파일:

- `src/stores/warehouseOutbound.js`

### 3.1 단일 읽기 모델 확장

- 공통 필드 정규화:
  - `outboundId`, `orderId`, `outboundType`, `sourceName`, `targetName`, `status`, `requestedAt`, `items`, `outboundStatusHistory`, `actionable`
- 상태 모델 통일:
  - `READY_TO_SHIP`
  - `IN_TRANSIT`
  - `COMPLETED`

### 3.2 source adapter 합성

- `STORE_OUTBOUND`: `storeOrders` 기반 매핑
- `WH_TRANSFER`: mock adapter
- `CIRCULAR_SALE`: mock adapter

### 3.3 필터/액션

- `activeTypeTab` 추가
- `getOutboundList(filters)`에서 유형/상태/기간/검색 동시 필터링
- `confirmOutbound()`는 `STORE_OUTBOUND`만 허용
- 비지원 유형은 `조회 전용` 메시지 반환

## 4. 출고 리스트 구현

파일:

- `src/views/warehouse/WarehouseOutboundView.vue`

반영 내용:

- 유형 필터 탭: `전체 / 매장 출고 / 창고간 이동 / 순환재고 판매`
- 상태 탭 유지: `전체 / 출고 준비중 / 배송중 / 완료`
- 기간 필터 유지: `전체 / 일 / 월 / 년` + 날짜 입력
- 검색 키워드 확장: `출고번호, 발주번호, 출고처, 목적지, 대표상품`
- 컬럼 통합:
  - 출고번호
  - 발주번호
  - 유형
  - 출고처
  - 목적지
  - 대표상품
  - 총 SKU
  - 총 수량
  - 상태
  - 요청일시

## 5. 출고 상세 구현

파일:

- `src/views/warehouse/WarehouseOutboundDetailView.vue`

반영 내용:

- 2단 패널 유지(좌 70 / 우 30)
- 우측 상단 상태 배지 + 진행 이력
- 액션 분기:
  - `STORE_OUTBOUND`: 출고확정 버튼 활성
  - `WH_TRANSFER`, `CIRCULAR_SALE`: 조회 전용 안내

## 6. 매장 입고 연계 경계

- `STORE_OUTBOUND` 상태 전이는 `storeOrders`를 source of truth로 유지한다.
- 출고 완료 의미는 매장 입고 확정(`RECEIVED`)과 동기화한다.
- `WH_TRANSFER`, `CIRCULAR_SALE`는 이번 단계에서 mock source 기반 조회만 제공한다.
