# 출입고 FE-BE 연동 구현보고서 (2026-05-10)

## 1. 구현 요약
- 창고 출고 화면을 더미 스토어 의존 구조에서 BE API 기반 조회/상태전이 구조로 전환했다.
- 매장 입고 화면을 더미 스토어 의존 구조에서 BE API 기반 조회/확정 구조로 전환했다.
- 출입고 연계 더미 로직을 제거하기 위해 `warehouseOutbound`, `storeOrder`의 출입고 전이 더미 경로를 정리했다.

## 2. 변경 파일
- `src/api/warehouse/outbound.js` (신규)
- `src/api/store/inbound.js` (신규)
- `src/views/warehouse/WarehouseOutboundView.vue` (교체)
- `src/views/warehouse/WarehouseOutboundDetailView.vue` (교체)
- `src/views/store/inbound/StoreInboundListView.vue` (교체)
- `src/views/store/inbound/StoreInboundDetailView.vue` (교체)
- `src/stores/warehouse/warehouseOutbound.js` (더미 제거/축소)
- `src/stores/store/storeOrder.js` (출입고 더미 전이 제거)
- `src/features/store/common/ui.js` (입고 상태 클래스 보강)

## 3. 핵심 연동 포인트
- 출고 API
  - `GET /api/warehouse/outbound`
  - `GET /api/warehouse/outbound/{outboundNo}`
  - `POST /api/warehouse/outbound/{outboundNo}/confirm`
  - `POST /api/warehouse/outbound/{outboundNo}/arrive`
- 입고 API
  - `GET /api/store/inbound`
  - `GET /api/store/inbound/{inboundNo}`
  - `POST /api/store/inbound/{inboundNo}/confirm`
- 공통 처리
  - `unwrap` 기반 `result` 추출
  - View `try-catch` + `extractErrorMessage` 사용자 메시지 처리
  - 액션 후 상세 재조회/목록 재조회로 상태 즉시 반영

## 4. 상태/버튼 정책 반영
- 출고 상세
  - `confirm`: `READY_TO_SHIP`에서만 활성
  - `arrive`: `IN_TRANSIT`에서만 활성
  - 호출 중 버튼 비활성화(`loadingAction`)
- 입고 상세
  - `confirm`: `PENDING_RECEIPT`에서만 활성
  - 호출 중 버튼 비활성화(`loadingAction`)

## 5. 더미 제거 범위
- 포함:
  - 출입고 화면 동작에 직접 연결된 더미 전이/가공 경로
  - `warehouseOutbound` mock 집계 경로
  - `storeOrder`의 출입고 상태 전이 더미 함수
- 제외:
  - 발주/분석 등 비출입고 도메인의 더미 데이터 구조(별도 브랜치)

## 6. 검증 결과
- `node --check` 성공
  - `src/api/warehouse/outbound.js`
  - `src/api/store/inbound.js`
  - `src/stores/warehouse/warehouseOutbound.js`
  - `src/stores/store/storeOrder.js`
- `npm run build` 성공
  - Vite production build 정상 완료
  - 대용량 chunk 경고만 존재(기능 오류 아님)

## 7. 후속 권장 작업
- 매장 입고 분석/이력 화면도 동일 API 기반 데이터소스로 통합 정리
- 출고 목록의 상세 메타(대표상품명 등)가 필요하면 BE 목록 DTO 확장 또는 FE 상세 프리패치 전략 검토
