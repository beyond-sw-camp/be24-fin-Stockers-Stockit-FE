# 순환재고 거래처 관리 구현 내역

## 1. 구현 개요

순환재고 판매 기능에서 사용하던 임시 거래처 더미 데이터를 판매 store 내부에서 분리하고, HQ `순환 재고 관리` 하위의 `순환 재고 거래처 관리` 기능을 신규 구현했다.

핵심 구현 방향:

- 순환재고 전용 거래처 store 추가
- HQ 거래처 관리와 분리된 별도 관리 화면 추가
- 판매 등록 페이지가 전용 거래처 목록을 참조하도록 연결
- AI 추천 기능은 제외하고 수기 등록/관리 중심의 MVP로 구현

## 2. 반영 내용

### 2.1 메뉴/라우트

추가 메뉴:

- `순환 재고 거래처 관리`

추가 라우트:

- `/hq/circular-inventory/buyers`

### 2.2 전용 거래처 store

파일:

- `src/stores/circularInventoryBuyers.js`

구현 내용:

- 순환재고 전용 거래처 seed 데이터 관리
- 거래처 등록/수정
- 업체명, 거래처 코드, 담당자명 기준 검색
- 상태 필터, 대표 소재 적합도 필터
- 산업군 선택 옵션 및 대표 소재 적합도 옵션 제공
- localStorage 기반 상태 유지

### 2.3 거래처 관리 화면

파일:

- `src/views/hq/circular-inventory/HqCircularInventoryBuyerManagementView.vue`

구현 내용:

- 좌측 거래처 목록 + 우측 등록/상세/수정 패널 구조
- 검색, 대표 소재 적합도 필터
- 신규 등록 모드와 선택 거래처 수정 모드
- 필수 입력값 검증

### 2.4 판매 등록 페이지 연동

관련 파일:

- `src/stores/circularInventory.js`
- `src/views/hq/circular-inventory/HqCircularInventorySalesRegisterView.vue`

변경 내용:

- 판매 store 내부 `buyers` 더미 배열 제거
- 순환재고 전용 거래처 store 참조로 변경
- 판매 등록 페이지 거래처 검색은 등록된 거래처 목록 전체를 참조
- 상단 선택 거래처 카드에 산업군, 대표 소재 적합도, 설명 표시
- 최종 판매 등록서 확인 모달에 거래처 설명과 대표 소재 적합도 표시
- 판매 저장 시 거래처 스냅샷 정보를 함께 저장

## 4. 검증 기준

- `순환 재고 관리 > 순환 재고 거래처 관리` 메뉴와 라우트가 노출되어야 한다.
- 신규 거래처 등록 시 필수값 검증이 동작해야 한다.
- 대표 소재 적합도는 단일 선택만 허용되어야 한다.
- 거래처 목록 검색과 상태/소재 필터가 동작해야 한다.
- 등록된 거래처를 선택하면 우측 상세/수정 패널에 값이 동일하게 반영되어야 한다.
- 등록된 거래처는 판매 등록 페이지에서 검색/선택 가능해야 한다.
- 판매 등록 페이지 상단 거래처 카드와 최종 확인 모달에 신규 거래처 정보가 표시되어야 한다.
