# 순환재고 판매 패널 단계형 전환 계획 및 구현 내역

## 1. 개요

기존 `거래처 선선택` 중심 플로우를 제거하고, 판매 패널 내부에서 `SKU 선택 -> 거래처 매칭 -> 판매 조건 확정` 순서로 진행하는 단계형 플로우로 전환했다.

핵심 목표:

- SKU 기준 선택(품목명 단위 일괄 선택 제거)
- 요청서 단위 소재 구분 단일성 보장
- 거래처 선택 시점의 후행 배치(Step 2)
- 판매 kg/단가의 운영 입력 명확화(Step 3)
- 최종 확인 모달에서 단계별 확정 정보 검토

---

## 2. 구현 계획(정리본)

### 2.1 단계 구조 재편

- Step 1: 판매 대상 SKU 선택
- Step 2: 선택 SKU 기반 거래처 선택(현 단계 수동)
- Step 3: SKU별 판매 kg/단가 확정

단계 완료 조건:

- Step 1 완료: SKU 1개 이상 + 소재 구분 단일성 충족
- Step 2 완료: 거래처 1개 선택
- Step 3 완료: 모든 SKU의 판매 kg > 0, 단가 > 0

### 2.2 소재 잠금 정책

- 요청서에서 첫 SKU 선택 시 `lockedMaterialType` 고정
- 다른 소재 구분 SKU는 선택 불가
- 전체 비우기 시 잠금 해제
- UI에서 비활성 상태를 명시적으로 표현

### 2.3 판매 수량/가격 정책

- 판매 kg: 관리자 직접 입력
- 실차감 수량: `Math.ceil(판매kg / SKU 단위중량)` (항상 올림)
- kg당 단가: SKU 기본 단가 자동 채움 + 수정 가능
- 금액: `판매kg * 단가`

### 2.4 화면 UX 정책

- 상단 거래처 검색 UI 제거, Step 2로 이동
- 단계 네비게이션은 카드형 + 진행 스트립(1/3 단위 fill)
- Step별 이전/다음 이동 제공
- 최종 확인 모달은 유지하되 단계 결과 반영

## 3. 구현 내역

### 3.1 스토어 상태/검증 확장

대상 파일:

- `src/stores/circularInventory.js`

주요 반영:

- 판매 draft를 SKU 단위로 관리하도록 정렬
- 단계 상태 `saleStep` 추가
- 요청서 소재 잠금 상태 `lockedMaterialType` 추가
- draft item 필드 확장:
  - `materialType`
  - `defaultKgUnitPrice`
  - `resolvedUnitPrice`
- 거래처 후보 필터링을 소재 구분 기준으로 연동
- 제출 검증 강화:
  - 소재 단일성 위반 차단
  - 단계 미완료(거래처 미선택/입력 미완료) 차단
  - SKU 및 품목 레벨 재고 초과 차단
  - 거래처 소재 적합성 검증

### 3.2 판매 등록 화면 단계형 전환

대상 파일:

- `src/views/hq/circular-inventory/HqCircularInventorySalesRegisterView.vue`

주요 반영:

- 판매 패널을 Step 1~3로 분리
- 단계 이동 가드 적용(선행 단계 미완료 시 차단)
- 기존 상단 거래처 선선택 UI 제거
- 거래처 선택 UI를 Step 2 내부로 이동

### 3.3 Step 1(SKU 선택) 개선

- SKU 기준 선택/제거 동작 정렬
- 소재 잠금에 따른 선택 불가 로직 반영
- 비활성 상태 UI 반영:
  - 회색 톤
  - 금지 커서
  - 안내 문구(`선택 불가`)
- 선택 SKU 테이블 컬럼 확장:
  - 소재 구분
  - 소재 상세
  - 재고 수량
  - kg당 단가
  - 환산 금액
  - 무게
  - 개당 무게

### 3.4 Step 2(거래처 매칭) 개선

- 검색 입력 + 후보 목록 수동 선택 UX 제공
- 선택 거래처 요약 정보를 우측 패널로 구성:
  - 업체명/코드
  - 담당자
  - 연락처
  - 산업군
  - 거래처 설명
- 불필요한 중복 박스 정리 및 레이아웃 폭 축소

### 3.5 Step 3(판매 조건 확정) 개선

- 컬럼 구성 정렬:
  - 품목
  - 소재 구분
  - 소재 상세
  - 현재 재고
  - 재고 총 kg
  - 판매 kg
  - 환산 수량
  - 실차감 수량
  - kg당 단가
  - 금액
  - 삭제
- 품목 제외 컬럼 가운데 정렬 정리
- 판매 kg 입력 UI 경량화:
  - 입력 박스 폭/높이 축소
  - 스피너 제거
  - `kg` 단위 라벨 분리 표시
- kg당 단가 UI 정리:
  - 기본은 읽기 모드
  - 수정 아이콘 클릭 시 입력 모드 전환
  - 입력 모드에서도 스피너 제거
- Step 3에서도 SKU 개별 삭제 지원
- Step 3 리스트 상단에 Step1과 동일한 `전체 비우기` 텍스트 액션 추가
- 요약 박스 라벨 변경:
  - `총 판매 KG` -> `요청 KG`
  - `총 금액` -> `예상 금액`
- 포함 소재를 우측 값 영역에 표시

### 3.6 단계 네비게이션/카피 정리

- 단계 버튼 라벨:
  - `SKU 선택`
  - `거래처 매칭`
  - `판매 조건 확정`
- 하단 액션 라벨:
  - `이전`
  - `다음`
- 확인 액션 라벨 변경:
  - `최종 판매 등록서 확인` -> `최종 거래 등록서 확인`
- 모달 타이틀도 동일 문구로 통일

## 4. 테스트 체크리스트

### 4.1 단계 전환

- SKU 미선택 시 Step 2 진입 차단
- 거래처 미선택 시 Step 3 진입/완료 차단
- 입력 미완료 시 최종 확인 차단

### 4.2 소재 잠금

- 첫 SKU 선택 시 `lockedMaterialType` 고정
- 다른 소재 SKU 선택 불가
- 같은 소재 SKU 다중 선택 가능
- 전체 비우기 후 잠금 해제

### 4.3 수량/차감/가격

- 판매 kg 입력 시 환산 수량 및 실차감(올림) 계산 일치
- 재고 초과 입력 시 제출 차단
- kg당 단가 기본값 자동 반영
- 단가 수정 시 금액 재계산

### 4.4 최종 확인

- 모달에 SKU/거래처/수량/단가/금액/소재 정보 노출 확인
- 최종 등록 후 판매 내역/재고 차감 반영 확인

---

## 5. 현재 가정 및 후속 확장

- 현재 Step 2는 수동 선택 전용(매칭 AI 미적용)
- 요청서 단위 소재 1종 고정 정책 유지
- 차감 정책은 `항상 올림` 유지
- 향후 AI 매칭 도입 시에도 Step 2 위치와 검증 계약은 유지하고, 추천 결과를 후보군 우선순위로 결합하는 방향을 권장
