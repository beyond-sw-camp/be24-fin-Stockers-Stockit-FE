# 순환재고 판매 구현 내역

## 1. 구현 개요

순환재고 판매 기능을 HQ `순환 재고 관리` 하위의 별도 기능으로 추가했다.  
기존 `순환 재고 조회` 페이지는 조회 전용으로 유지하고, 실제 판매 작업은 `순환 재고 판매 등록` 화면에서 수행하도록 분리했다.

핵심 구현 방향:

- 순환재고 전용 store 추가
- 순환재고 조회 데이터와 판매 데이터를 동일한 store에서 관리
- 거래처 검색, 하단 슬라이딩 판매 패널, 판매 내역, 판매 분석 화면 추가
- `kg 입력 -> 참고 벌 수량 계산 -> 실제 벌 수량 올림 차감` 규칙 반영

## 2. 반영 내용

### 2.1 메뉴/라우트

HQ `순환 재고 관리` 하위 메뉴에 아래 항목을 추가했다.

- `순환 재고 판매 등록`
- `순환 재고 판매 내역`
- `순환 재고 판매 분석`

추가 라우트:

- `/hq/circular-inventory/sales/register`
- `/hq/circular-inventory/sales/history`
- `/hq/circular-inventory/sales/analysis`

### 2.2 전용 store

파일:

- `src/stores/circularInventory.js`

구현 내용:

- 순환 재고 seed 데이터 전용 관리
- 순환재고 전용 거래처 더미 데이터셋 추가
- 판매 draft 상태 관리
- 판매 등록 후 판매 이력 저장
- localStorage 기반 재고/판매 상태 유지
- 판매 등록 시 재고 차감 반영

주요 규칙:

- 판매 입력 기준은 `kg`
- 참고 벌 수량은 `판매 kg / 단위중량`
- 실제 재고 차감은 `Math.ceil(참고 벌 수량)`

### 2.3 조회 화면 정리

파일:

- `src/views/hq/circular-inventory/HqCircularInventoryView.vue`

변경 내용:

- 조회 화면을 조회 전용으로 정리
- 기존 판매 버튼/선택 UI 제거
- 순환재고 전용 store를 사용하도록 변경
- 상단에 `판매 등록은 별도 메뉴에서 진행` 안내 문구 추가

### 2.4 판매 등록 화면

파일:

- `src/views/hq/circular-inventory/HqCircularInventorySalesRegisterView.vue`

구현 내용:

- 상단 거래처 검색/선택 영역
- 기존 조회 테이블 기반 판매 대상 리스트
- 각 행 `판매 패널에 추가` 버튼
- 하단 슬라이딩 판매 등록 패널
- 패널 내 kg 입력, 참고 벌 수량, 실제 차감 벌 수량, 단가, 금액 계산
- 거래처 미선택/단가 미입력/판매 kg 미입력 상태에서는 등록 차단

### 2.5 판매 내역/분석 화면

파일:

- `src/views/hq/circular-inventory/HqCircularInventorySalesHistoryView.vue`
- `src/views/hq/circular-inventory/HqCircularInventorySalesAnalysisView.vue`

구현 내용:

- 판매 내역 목록 + 상세 패널
- 거래처/품목/판매번호 검색
- 카테고리별, 거래처별, 소재별 판매 집계

## 4. 검증 기준

- 판매 등록 화면 진입 시 기존 순환 재고 조회 필터/정렬/목록이 정상 동작해야 한다.
- 거래처 검색 후 1건 선택하면 상단 거래처 정보가 유지되어야 한다.
- `판매 패널에 추가` 버튼 클릭 시 하단 패널에 품목이 추가되어야 한다.
- 같은 품목을 다시 추가해도 중복 줄이 생기지 않고 기존 줄을 수정 대상으로 사용해야 한다.
- 판매 kg 입력 시 참고 벌 수량과 금액이 즉시 계산되어야 한다.
- 환산 벌 수량이 정수가 아니면 올림 차감 규칙이 UI와 등록 결과에 일관되게 반영되어야 한다.
- 단가가 비어 있으면 등록되지 않아야 한다.
- 거래처 미선택 상태에서는 등록되지 않아야 한다.
- 판매 등록 완료 시 판매 내역/분석 화면에서 결과가 반영되어야 한다.
- 판매 등록 후 순환 재고 조회 화면에서 재고 수량/무게가 감소한 상태로 보여야 한다.
