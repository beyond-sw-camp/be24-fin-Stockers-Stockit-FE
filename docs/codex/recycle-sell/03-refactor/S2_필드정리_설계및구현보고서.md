# 순환재고 판매 S2 필드 정리 설계/구현 보고서
작성일: 2026-05-20  
대상: Stockit-FE `circularStock` 판매 데이터 경로

## 1. 작업 목적
S2 단계의 목표는 판매 상세/목록 데이터에서 중복/별칭 파생 필드를 최소화하고, BE 응답 기준의 단일 필드 체계로 정리하는 것이다.

이번 라운드에서 확정한 기준:
- 상세 합계 UI는 단일 총액/총수량 중심으로 단순화
- 아이템 명칭 표준은 `productName`으로 고정
- 숫자 의미가 동일한 중복 필드는 store 상세 상태에서 제거

## 2. 설계 기준
### 2.1 필드 정책
- 유지 표준(핵심):
  - `saleId`, `saleNo`, `outboundStatus`, `totalSoldQuantity`, `totalAmount`
- 상세 렌더에 필요한 보조 필드 유지:
  - `soldAt`, `buyerName`, `materialType`, `headline`, `totalSkuCount`,
    `totalRequestedWeightKg`, `totalActualWeightKg`, `soldByName`, `items`, `statusHistory`

### 2.2 제거 정책
- 판매 상세 중복 필드 제거:
  - `totalItems` (=`totalSkuCount`)
  - `totalDeductedQuantity` (=`totalSoldQuantity`)
  - `totalActualAmount`/`totalRequestedAmount` (둘 다 `totalAmount`)
  - `soldBy` (=`soldByName`)
- 상세 아이템 중복 필드 제거:
  - `itemName` (=`productName`)
  - `deductedQuantity` (=`soldQuantity`)
  - `requestedAmount`/`actualAmount` (둘 다 `lineAmount`)

### 2.3 구현 원칙
- 매퍼는 “1개 의미 = 1개 필드” 원칙 준수
- 뷰 표시용 파생은 템플릿/계산으로 처리하고, store 상세 상태에 중복 저장하지 않음
- 등록(draft) 도메인은 S2 범위에서 구조 유지(회귀 방지), 상세 응답 경로만 우선 정리

## 3. 구현 내역
## 3.1 스토어 변경
대상 파일:
- `src/stores/hq/circularStock/circularStock.js`

주요 변경:
1. `mapSaleDetailFromApi()` 정리
- 상세 루트에서 중복 필드 제거:
  - 삭제: `totalItems`, `totalDeductedQuantity`, `totalActualAmount`, `totalRequestedAmount`, `soldBy`
- 상세 아이템에서 중복 필드 제거:
  - 삭제: `itemName`, `deductedQuantity`, `requestedAmount`, `actualAmount`
- 유지:
  - 아이템: `productName`, `soldQuantity`, `lineAmount` 등

2. 내부 합계 참조 표준화
- ESG 계산 매출 fallback:
  - `totalAmount` 및 `lineAmount` 기준으로 정리
- analytics 합계:
  - 판매 합계는 `sale.totalAmount`
  - 카테고리 합계는 `item.lineAmount`

3. 제출 payload 금액 참조 보정
- `lineAmount: Number(item.lineAmount || 0)`으로 정리

## 3.2 상세 화면 변경
대상 파일:
- `src/views/hq/circular-stock/HqCircularStockSalesDetailView.vue`

주요 변경:
1. 헤더/요약 바인딩 치환
- `sale.totalItems` → `sale.totalSkuCount`
- `sale.totalDeductedQuantity` → `sale.totalSoldQuantity`
- `sale.totalActualAmount`/`sale.totalRequestedAmount` → `sale.totalAmount`

2. 상세 SKU 라인 바인딩 치환
- `item.itemName` → `item.productName`
- `item.deductedQuantity` → `item.soldQuantity`
- `item.requestedAmount`/`item.actualAmount` → `item.lineAmount`

3. 중복 표시 단순화
- 수량 카드: 단일 `totalSoldQuantity`
- 금액 카드: 단일 `totalAmount`
- SKU 테이블 금액: `lineAmount` 단일 표기

## 4. 변경 전/후 요약
### 4.1 상세 루트 필드
- Before:
  - `totalSkuCount`, `totalItems`
  - `totalSoldQuantity`, `totalDeductedQuantity`
  - `totalAmount`, `totalActualAmount`, `totalRequestedAmount`
  - `soldByName`, `soldBy`
- After:
  - `totalSkuCount`, `totalSoldQuantity`, `totalAmount`, `soldByName`만 유지

### 4.2 상세 아이템 필드
- Before:
  - `productName`, `itemName`
  - `soldQuantity`, `deductedQuantity`
  - `lineAmount`, `requestedAmount`, `actualAmount`
- After:
  - `productName`, `soldQuantity`, `lineAmount`만 유지
