# 순환재고 거래처 관리 설계안

## 1. 개요

순환재고 판매 기능에서 사용하던 임시 거래처 더미 데이터를 분리하고, HQ `순환 재고 관리` 하위의 전용 거래처 관리 기능을 추가한다.

핵심 방향:

- 기존 HQ `거래처 관리`와 분리된 별도 도메인으로 관리
- 거래처 등록/조회/수정을 한 화면에서 처리
- 판매 등록 페이지는 이 전용 거래처 목록을 참조
- AI 추천 기능은 이번 범위에서 제외

## 2. 거래처 모델

필드:

- `id`
- `code`
- `companyName`
- `industryGroup`
- `productTypes`
- `description`
- `primaryMaterialFit`
- `managerName`
- `phone`
- `createdAt`
- `updatedAt`

대표 소재 적합도는 아래 3개 중 단일 선택으로 고정한다.

- `natural-single`
- `synthetic`
- `blended`

산업군은 문서 기반 선택형 옵션을 제공한다.

- 천연 단일 관련: 재생원사, 홈텍스타일, 반려동물 용품, 패션 잡화, 의료/위생, 인테리어, 음향 기기, 농업, 뷰티/화학, 교육/공예
- 합성 관련: 단체복/워크웨어, 아웃도어 잡화, 물류 자재, 해양/수산, 구호 용품, 가구 제조, 자동차 부품, 레저/기어, 화학 사출, 생활 잡화
- 혼방 관련: 의류/워크웨어, 유니폼, 건설 자재, 자동차 흡음, 산업 소모품, 가구 자재, 에너지, 물류 패키징, 농업, 반려동물

## 3. 화면 구조

메뉴:

- `순환 재고 거래처 관리`

화면 구성:

- 좌측 목록 패널
  - 거래처 검색
  - 상태 필터
  - 대표 소재 적합도 필터
  - 거래처 목록
- 우측 상세/등록 패널
  - 신규 등록 모드
  - 선택 거래처 수정 모드
  - 상태 전환 버튼

목록 최소 노출 항목:

- 업체명
- 산업군
- 대표 소재 적합도
- 담당자명
- 상태

등록/수정 필드 순서:

- 업체명
- 거래처 코드
- 산업군
- 취급 제품/생산품
- 거래처 설명
- 대표 소재 적합도
- 담당자명
- 연락처
- 상태

## 4. 판매 등록 연동

판매 등록 페이지 변경 방향:

- 기존 `buyers` 더미 배열 제거
- 신규 순환재고 거래처 store 참조
- 상단 거래처 카드와 최종 확인 모달에 아래 정보 표시
  - 업체명
  - 거래처 코드
  - 담당자명
  - 연락처
  - 거래처 설명
  - 대표 소재 적합도

## 5. 테스트 기준

- 메뉴와 라우트가 정상 노출되어야 한다.
- 거래처 필수값 검증이 동작해야 한다.
- 대표 소재 적합도는 단일 선택만 가능해야 한다.
- 거래처 검색은 업체명, 코드, 담당자명 기준으로 동작해야 한다.
- 상태/대표 소재 적합도 필터가 동작해야 한다.
- 등록된 거래처는 판매 등록 페이지에서 정상 검색/선택되어야 한다.
- 판매 등록 페이지 거래처 카드와 최종 확인 모달에 신규 거래처 정보가 표시되어야 한다.
