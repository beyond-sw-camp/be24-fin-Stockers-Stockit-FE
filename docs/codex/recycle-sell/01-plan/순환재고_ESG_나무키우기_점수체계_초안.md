# 순환재고 ESG 나무키우기 점수체계 초안

## 1. 문서 목적

본 문서는 `순환 재고 판매 1건`을 기준으로 산정되는 `ESG 나무키우기 점수`의 초안을 정리한 보고서다.

현재 기준의 목적은 다음과 같다.

- 판매 1건이 만들어낸 환경 성과를 점수와 KPI로 분리해 설명한다.
- 순환 재고 판매 상세 페이지에서 한눈에 읽히는 구조를 만든다.
- 이후 ESG 대시보드 반영 논의를 위한 기준 초안을 남긴다.

주의:

- 본 점수체계는 현재 `순환 재고 판매 상세 페이지`와 `순환 재고 판매 저장 스냅샷`에만 적용한다.
- `ESG 대시보드`에는 아직 적용하지 않는다.
- 본 문서는 운영 확정안이 아니라 `현재 구현 및 논의 기준 초안`이다.

## 2. 기본 원칙

### 2.1 점수와 KPI를 분리한다

`ESG 나무키우기 점수`는 여러 점수 요소의 합이다.

- 점수: 환경 활동과 추적 완결성에 대한 평가값
- KPI: 판매 성과와 환경 성과를 설명하는 참고 수치

즉, 아래처럼 본다.

- `나무키우기 점수 = 점수 요소 합계`
- `탄소 감축량 / 탄소 가치 / 판매 수익 / 거래 매출 = KPI`

### 2.2 판매 1건 단위 스냅샷으로 저장한다

수치는 판매 등록 시점에 계산해서 판매건에 저장한다.

이렇게 하는 이유:

- 과거 판매건 수치가 이후 시세나 계수 변경으로 흔들리지 않게 하기 위함
- 판매 상세에서 당시 기준의 ESG 성과를 그대로 복원하기 위함

기존 판매 이력처럼 스냅샷이 없는 경우에는 상세 페이지에서 `추정치`로 fallback 계산한다.

### 2.3 환경 중심으로 해석한다

이번 점수체계는 `순환 판매 실행`, `자원 순환`, `탄소 절감`, `추적 가능성`을 중심으로 본다.

아직 반영하지 않는 항목:

- 업사이클 파트너십 점수
- 고부가 순환 전환 점수
- ESG 대시보드 누적 점수 연동

## 3. 점수 구조

현재 점수는 아래 4개 요소로 구성한다.

### 3.1 순환 판매 실행 점수

- 의미: 판매 1건이 실제로 최종 등록 완료되었는가
- 성격: 고정 이벤트 점수
- 현재 기준: `120pt`

공식:

`판매 품목이 1개 이상 존재하면 기본 120pt 부여`

해석:

- 점수가 높다는 의미보다, “실제 순환 판매 행동이 일어났다”는 사실을 기록하는 점수다.

### 3.2 자원 순환 전환 점수

- 의미: 폐기 가능성이 있던 재고가 실제 순환 자산으로 전환된 정도
- 성격: 소재별 무게 기반 점수
- 계산 단위: `실제 판매 반영 무게 × 소재별 순환 전환 계수`

혼합 소재는 비율대로 분해해 계산한다.

예:

- 면 70 / 폴리 30 혼합 소재 10kg 판매 시
- 면 7kg, 폴리 3kg로 분리해 각각 계산 후 합산

해석:

- 탄소 절감과 별개로, “재고를 다시 순환 흐름으로 넣었다”는 자원 관점의 기여분이다.

### 3.3 탄소 절감 기여 점수

- 의미: 실제 탄소 감축량이 얼마나 발생했는가
- 성격: 환경 성과 핵심 점수
- 현재 기준 배율: `1kgCO₂ = 1pt`

공식:

`실제 탄소 감축량(kgCO₂) × 1`

현재 구현 상수:

- `CARBON_POINT_MULTIPLIER = 1`

해석:

- 가장 직관적으로 환경 성과와 연결되는 점수다.
- 상세 페이지에서도 이 점수가 가장 크게 읽히도록 시각화하는 것을 기본 원칙으로 삼는다.

### 3.4 인증 / 추적 완료 점수

- 의미: 판매건의 ESG 근거 데이터가 얼마나 완성도 있게 남았는가
- 성격: 데이터 신뢰도 보정 점수
- 계산 방식: 항목별 rule 기반 합산

현재 확인 기준 예시:

- 소재 분류 완료 여부
- 거래처 유형 확인 여부
- 처리 방식 결정 가능 여부
- 탄소 계산 근거 저장 여부
- 판매 스냅샷 저장 여부

해석:

- 환경 성과 자체보다, 그 성과를 설명하고 감사 대응할 수 있는 준비 수준을 반영한다.

## 4. KPI 구조

현재는 아래 KPI를 함께 저장하고 보여준다.

### 4.1 이번 판매 탄소 감축량

- 의미: 이번 판매로 인해 감축된 실제 탄소량
- 단위: `kgCO₂`

상세 페이지 문구:

- `얼마나 탄소를 줄였나`

### 4.2 save한 탄소배출권 환산 가치

- 의미: 탄소 감축량을 판매 시점 탄소배출권 가치로 환산한 참고 수치
- 기준: 판매 시점 `KAU 가격`

상세 페이지 문구:

- `탄소 가치로 환산하면`

### 4.3 거래 가능 탄소배출권 추정치

- 의미: 거래 가치까지 확장해서 본 참고 수치
- 현재 기준: MVP 단계에서는 `탄소배출권 환산 가치`와 동일 값 사용

### 4.4 순환 재고 판매 수익

- 의미: 폐기 손실이 판매 수익으로 전환된 금액
- 현재 기준: `wasteLossRecoveredValue = salesRevenue`

상세 페이지 문구:

- `얼마나 이익이 났나`

### 4.5 실제 거래 매출

- 의미: 판매 확정 금액 기준의 실제 매출
- 상세 페이지에서는 참고 KPI로 함께 보여준다.

## 5. 현재 구현 산식 요약

현재 `src/stores/circularStock.js` 기준의 주요 상수는 아래와 같다.

- `EXECUTION_BASE_POINTS = 120`
- `CARBON_POINT_MULTIPLIER = 1`

점수 스냅샷 구조:

- `scoreBreakdown[]`
  - `execution`
  - `resourceCirculation`
  - `carbonContribution`
  - `traceability`
- `kpiSnapshot`
  - `savedCarbonKg`
  - `carbonCreditValue`
  - `tradableCarbonCreditValue`
  - `salesRevenue`
  - `wasteLossRecoveredValue`
- `esgMeta`
  - `treeGrowPoints`
  - `treatmentType`
  - `kauPriceAtSale`
  - `formulaSummary`
  - `materialBreakdown[]`
  - `traceabilityBreakdown[]`

핵심 계산 흐름:

1. 판매 품목별 실제 반영 무게를 기준으로 소재를 분해한다.
2. 소재별 순환 전환 계수와 탄소 절감 계수를 적용한다.
3. 점수 요소별 점수를 계산한다.
4. KPI 수치를 계산한다.
5. 판매 등록 시점 스냅샷으로 저장한다.

## 6. 상세 페이지 시각화 원칙

현재 상세 페이지는 숫자를 나열하는 방식이 아니라 `해석 중심 UI`로 정리한다.

### 6.1 판매 상세 탭

거래 중심 화면이므로 ESG는 요약만 보여준다.

구성:

- `Impact Snapshot`
- `ESG 나무 키우기 점수`
- `점수 요소`
- `KPI 지표`

표현 원칙:

- 숫자만 보여주지 않고 `무엇을 의미하는지` 짧은 설명을 붙인다.
- 점수 요소는 막대형 비중으로 보여줘서 어느 요소가 큰지 바로 읽히게 한다.
- KPI는 `얼마나 이익이 났나`, `얼마나 탄소를 줄였나`처럼 질문형 제목으로 읽기 쉽게 만든다.

### 6.2 ESG 성과 탭

ESG와 환경 관점의 상세 설명을 별도 탭으로 분리한다.

구성:

- `Environmental Story`
- 점수 구조 요약
- 점수 요소 상세
- KPI 지표 상세
- 점수 / KPI 산정 근거
- 인증 / 추적 완료 점수 상세
- 소재별 자원 순환 반영 상세
- 소재별 탄소 절감 반영 상세

표현 원칙:

- 사용자가 “이 판매건이 환경적으로 무슨 의미가 있었는지”를 먼저 읽게 한다.
- 이후에 “그 의미가 어떤 점수와 수치로 계산되었는지”를 따라가게 한다.

## 7. 현재 해석 문구 기준

상세 페이지에서 점수와 KPI는 아래 같은 해석 문구를 사용한다.

점수 요소 예시:

- 순환 판매 실행 점수: `판매 행동 자체가 순환 활동으로 인정된 점수`
- 자원 순환 전환 점수: `폐기 예정 재고를 다시 순환 자산으로 전환한 기여분`
- 탄소 절감 기여 점수: `실제 탄소 감축량과 직접 연결되는 핵심 환경 점수`
- 인증 / 추적 완료 점수: `근거 데이터와 추적 정보가 얼마나 잘 남았는지 보여주는 점수`

KPI 예시:

- `얼마나 이익이 났나`
- `얼마나 탄소를 줄였나`
- `탄소 가치로 환산하면`
- `실제 거래 매출`

## 8. 향후 검토 과제

본 초안은 지금까지의 구현과 논의를 정리한 기준점이며, 아래 항목은 후속 검토가 필요하다.

### 8.1 점수 보정

- 실행 점수 `120pt`가 적절한지
- 탄소 절감 점수 배율 `1kgCO₂ = 1pt`가 적절한지
- 자원 순환 전환 점수의 소재별 계수 가중치가 적절한지

### 8.2 KPI 해석 보정

- `거래 가능 탄소배출권 추정치`를 별도 항목으로 계속 둘지
- `순환 재고 판매 수익`과 `실제 거래 매출`을 어떻게 구분해서 운영할지

### 8.3 확장 후보

후속 후보로는 아래가 남아 있다.

- 업사이클 파트너십 점수
- 고부가 순환 전환 점수

단, 현재는 미적용 상태이며 본 초안의 점수 합산에도 포함하지 않는다.

### 8.4 대시보드 연동

- 현재는 `순환 재고 판매 상세`에만 반영했다.
- ESG 대시보드 연동 여부는 별도 의사결정이 필요하다.
- 현 단계에서는 `대시보드 미반영`을 유지한다.

## 9. 결론

현재 초안의 핵심은 다음 세 줄로 요약할 수 있다.

- `나무키우기 점수`는 판매 1건의 환경 활동 점수 합이다.
- `탄소 감축량, 탄소 가치, 판매 수익, 거래 매출`은 KPI로 분리해 본다.
- 상세 페이지는 숫자를 나열하는 것이 아니라, `성과 해석 -> 점수 구조 -> 산정 근거` 순으로 읽히게 만든다.

이 문서는 향후 운영 기준 확정 전까지 `순환 재고 판매 ESG 점수체계`의 기준 초안으로 사용한다.
