# 순환재고 판매 페이지 1차 설계안

## 1. 요약

기존 `순환 재고 조회` 화면을 기반으로 활용하되, 실제 판매 작업은 별도 라우트의 `판매 전용 화면`에서 수행한다.  
화면 구조는 `상단 거래처 검색/입력 + 중앙 순환재고 조회 테이블 + 하단 슬라이딩 판매 등록 패널` 방식으로 구성한다.

핵심 방향:

- 조회와 판매를 한 화면 안에서 이어서 처리하되, 기존 HQ 순환 재고 조회와 일반 매장 판매는 도메인을 분리한다.
- 기존 순환 재고 조회 페이지는 백업/비교 목적까지 고려해 메뉴에 계속 노출한다.
- “조회 전용”과 “판매 등록”의 목적을 메뉴명과 화면 헤더에서 명확히 구분한다.
- 현재 조회 페이지는 복제해서 판매 화면으로 출발하되, 이후 공통 컴포넌트 분리 리팩터링을 후속 과제로 둔다.

## 2. 메뉴/라우트 구조

HQ `순환 재고 관리` 하위 메뉴:

- `순환 재고 후보 조회`
- `순환 재고 조회`
- `순환 재고 판매 등록`
- `순환 재고 판매 내역`
- `순환 재고 판매 분석`

라우트 기준:

- `/hq/circular-inventory/candidates`
- `/hq/circular-inventory`
- `/hq/circular-inventory/sales/register`
- `/hq/circular-inventory/sales/history`
- `/hq/circular-inventory/sales/history/:saleId`
- `/hq/circular-inventory/sales/analysis`

## 3. 판매 등록 화면 레이아웃

### 3.1 상단 고정 영역

- 페이지 제목: `순환 재고 판매 등록`
- 설명 문구: 조회 리스트에서 판매할 순환 재고를 선택해 등록하는 화면임을 안내
- 거래처 검색/선택 필드
- 선택 거래처 요약 카드
- 판매 패널 열기 버튼

### 3.2 중앙 본문

- 기존 순환 재고 조회 필터/테이블 유지
- 체크박스 기반 선택은 사용하지 않음
- 각 행에 `판매 패널에 추가` 버튼 제공
- 이미 패널에 담긴 항목은 `수정` 상태로 표시

### 3.3 하단 슬라이딩 패널

- 기본은 접힘
- 항목이 추가되면 열 수 있는 요약 바 노출
- 펼치면 판매 등록 폼과 선택 품목 리스트를 함께 표시
- 닫혀 있어도 `거래처명 / 담긴 품목 수 / 총 판매 kg / 등록 버튼` 수준의 요약 정보는 유지

## 4. 판매 패널 동작 규칙

공통 입력:

- 거래처 선택
- 메모

품목별 입력:

- 품목 코드
- 품목명
- 카테고리
- 소재 구성
- 현재 순환 재고 수량(벌)
- 현재 환산 재고(kg)
- 판매 kg 입력
- 참고 벌 수량 표시
- 단가 입력
- 금액 계산 결과
- 항목 제거 버튼

추가/수정 규칙:

- 테이블에서 항목을 선택하는 방식이 아니라 `판매 패널에 추가` 버튼으로 담는다.
- 같은 항목을 다시 추가하면 새 줄을 만들지 않고 기존 줄을 수정 대상으로 사용한다.
- 판매 패널이 source of truth가 된다.
- 판매 등록 버튼은 패널 하단에 고정한다.

## 5. 수량/차감 규칙

- 사용자가 입력하는 기준값은 `kg`
- `벌 수량`은 참고값으로 표시
- 참고 벌 수량 계산은 `판매 kg / 단위중량`
- 표시용 참고 벌 수량은 소수점 포함 값으로 최대한 정확하게 보여준다.
- 실제 재고 차감 기준은 `벌 수량 올림 차감`
  - 예: 환산 결과가 `3.2벌`이면 `4벌 차감`
- 판매 등록은 `kg 입력 -> 참고 벌 수량 계산 -> 올림 차감` 규칙으로 처리한다.
- 화면에는 아래 안내 문구를 명시한다.
  - `판매 kg 기준으로 등록하며, 실제 재고 차감은 환산 벌 수량 기준 올림 처리됩니다.`

## 6. 거래처 모델

거래처는 기존 vendor와 분리된 `순환재고 전용 거래처` 모델을 둔다.

1차 범위:

- 전용 store 또는 더미 데이터셋으로 검색 가능하게 구현
- 검색 기준:
  - 거래처명
  - 거래처 코드
  - 담당자명
- 선택 결과:
  - 거래처 ID
  - 거래처명
  - 담당자명
  - 연락처
  - 비고

## 8. 조회 화면과 판매 화면의 관계

초기 구현은 `기존 페이지 복제`로 진행한다.

- 조회 화면은 조회만 수행한다.
- 판매 화면은 조회 UI를 포함하지만 판매 행동까지 가진다.
- 두 파일 간 공통 마크업이 커지면 2차 작업에서 공용 컴포넌트로 분리한다.

메뉴에 둘 다 노출할 때의 주의사항:

- 조회 화면에는 판매 CTA를 두지 않는다.
- 판매 화면에는 거래처 선택, 판매 패널, 등록 흐름이 보여야 한다.
- 상단 헤더 문구를 완전히 다르게 두어 사용 목적을 분리한다.

## 9. 판매 내역 / 상세 UX

- `순환 재고 판매 내역`은 판매건 헤더 목록 전용 화면으로 사용한다.
- 목록 표시 단위는 `최종 등록 1회 = 판매건 1건` 기준이다.
- 목록 행 클릭 시 `판매 상세 페이지`로 이동한다.
- 판매 상세는 별도 라우트(`/hq/circular-inventory/sales/history/:saleId`)에서 조회한다.
- 상세 페이지는 판매번호, 판매일시, 거래처, 등록자, 메모, 요청/실제 KG, 요청/실제 금액, 품목 라인을 보여준다.
- 판매 등록에서 최종 등록이 완료되면 생성된 `saleId` 상세 페이지로 즉시 이동한다.
- 상세 조회는 별도 신규 상태 모델 없이 기존 `CircularInventorySale`와 `getSaleById(saleId)` 조회 방식만 재사용한다.
