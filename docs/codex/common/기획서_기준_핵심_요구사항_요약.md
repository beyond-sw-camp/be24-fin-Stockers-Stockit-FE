## 1. 프로젝트 정체성

`STOCKIT`은 SPA 브랜드와 패션 리테일 기업을 위한 ESG 스마트 재고관리 시스템이다.  
핵심은 단순 재고 수량 관리가 아니라, 다음 4가지를 하나의 플랫폼에서 통합하는 것이다.

- 매장-창고-본사 통합 재고 운영
- 데이터 기반 수요 예측과 발주 의사결정
- 악성 재고의 순환 자산 전환
- ESG 성과의 정량화와 시각화

즉, 이 프로젝트는 일반 ERP 프론트엔드가 아니라 `패션/의류 도메인 기반 SCM + ESG 운영 시스템`으로 이해해야 한다.

## 2. 타겟 사용자와 역할

기획서의 역할은 3개다.

- `CentralManager`: 본사 관리자
- `StoreManager`: 매장 관리자
- `WarehouseManager`: 물류창고 관리자

각 역할의 기대 업무는 아래와 같다.

- 본사 관리자: 전사 재고 관제, 제품/거래처 관리, 창고 간 재고 이동 지시, 순환 재고 관리, ESG 지표 모니터링
- 매장 관리자: 재고 조회, 판매, 발주 요청, 입고 처리, 통계 조회
- 물류창고 관리자: 입출고 처리, 피킹 리스트 관리, 창고 재고 및 운영 진척 모니터링

## 3. MVP 핵심 기능

기획서의 1순위 MVP는 `매장·OMS·WMS 핵심 CRUD + 역할 기반 인증/인가 + 발주/입출고 기본 흐름`이다.

화면/기능 수준으로 보면 다음 항목이 우선이다.

### 3.1 매장 관리자

- 실시간 매장 재고 조회
- 창고에서 들어온 물품 입고 검수 및 재고 반영
- 판매 입력 및 재고 차감
- 물류창고 대상 발주 요청
- 발주 상태 조회, 수정, 취소
- 일/월/년 매출 통계
- 상품별 판매량 및 발주량 통계

### 3.2 본사 관리자

- 상품 및 거래처 관리
- 매장 목록 및 매장별 상세 재고 조회
- 창고 목록 및 창고별 상세 재고 조회
- 전사 통합 재고 조회
- 전사 재고 현황판
- 재고 부족 알림 및 공지 관리
- 전국 매장 발주량과 창고 가용량 비교
- 창고 간 재고 이동 지시

### 3.3 물류창고 관리자

- 협력사 → 창고 입고 처리
- 매장 발주 기반 피킹 리스트 확인
- 피킹 적재 체크
- 실시간 창고 재고 현황 모니터링
- 공간 점유율 확인
- 입고/출고 작업 진척도 확인
- 출고 처리 및 창고 재고 차감

## 4. 시스템 자동화 요구사항

기획서에서 자동화는 단순 부가 기능이 아니라 핵심 방향이다.

- 매장 발주 자동/반자동 승인
- 피킹 리스트 자동 생성
- 배송 리스트 자동 지정
- 기사 매핑 및 자동 배차
- 안전재고 기반 발주 권장 리스트 생성
- AI 통계 분석과 수요 예측 시각화

발주 권장 로직의 핵심 공식은 아래와 같다.

`(현재 창고 재고 + 입고 예정 물량) - (매장 발주 예약 물량) < 안전 재고`

위 조건을 만족하면 부족분만큼 생산처 발주 권장 리스트를 생성한다.

## 5. 핵심 차별화 기능

기획서에서 가장 중요한 차별화 기능은 아래 4가지다.

### 5.1 순환 재고 시스템

- 장기 미판매, 판매율 저조, 사이즈 파편화 재고를 후보로 자동 추출
- 정상 판매 재고와 분리해 `순환 재고`로 전환
- 소재/함량/총 무게 기준으로 재분류
- 업사이클링/다운사이클링/재사용/원단 환원 거래처에 판매 등록

### 5.2 ESG 친환경 발자국 현황판

- 탄소 배출 절감량
- 소재 재활용 전환율
- 폐기물 감소량
- 폐기 손실 수익 전환액

또한 ESG 점수 누적에 따라 10단계 나무 성장 시각화가 포함된다.

점수 공식:

`점수 = 재활용되는 의류 총 무게(kg) × 소재별 탄소 환산 계수(kgCO2/kg)`

### 5.3 물류창고 간 재고 이동

- 창고 간 과다/부족 재고 비교
- 출고 창고, 입고 창고, 품목, 수량 지정
- 이동 예정 재고 격리
- 배송 중 상태 추적
- 도착 창고 입고 확인 후 가용 재고 반영

### 5.4 AI 스마트 리포트

- 핵심 브리핑 자동 요약
- 수요 예측
- 진열 추천
- 연관 상품 추천

리포트는 `매장 AI 리포트`와 `본사 AI 리포트` 두 종류가 명시돼 있다.

## 6. 도메인 기준

프로젝트의 기본 도메인은 `SPA 패션/의류`다.  
따라서 재고와 순환 기능의 주요 판단 기준도 의류 특성을 반영해야 한다.

- 시즌성
- 사이즈 분포
- 컬러 잔량 편중
- 소재 구성
- 의류 무게
- 업사이클링/리사이클링 활용 가능성

즉, 단순 공산품 재고관리 모델이 아니라 `의류 라이프사이클 관리`가 핵심이다.

## 7. 기술/아키텍처 방향

기획서 기준 기술 방향은 아래와 같다.

- 프론트엔드: Vue.js
- 백엔드: Java, Spring Boot, Spring Data JPA
- DB: MariaDB, Redis
- 메시징: Kafka
- 검색/파이프라인: Elasticsearch, Logstash
- 인프라: Docker, Kubernetes, Jenkins, AWS
- AI: OpenAI ChatGPT API
- 아키텍처: MSA

프론트 작업 시에도 이 구조를 염두에 두고, 단순 단일 앱 데모가 아니라 추후 MSA 백엔드와 붙을 수 있는 화면/상태/용어 설계를 유지하는 것이 중요하다.

## 8. 비기능 요구사항

기획서에 명시된 주요 비기능 요구사항은 다음과 같다.

- 재고 조회 및 발주 처리 API 응답 시간 평균 200ms 이하
- 동시 접속 1,000명 이상
- Redis/Kafka 기반 동시성 및 비동기 처리
- Elasticsearch 기반 검색 성능 강화
- RBAC 기반 권한 분리
- JWT 기반 인증/인가
- HTTPS
- 민감 데이터 암호화 저장
- CI/CD 자동화
- 블루/그린 또는 무중단 배포 고려

## 9. 당분간 작업 원칙

기획서가 바뀌기 전까지는 아래 원칙을 우선 적용한다.

- 화면/기능 명칭은 기획서 용어를 우선 사용한다.
- 제품/재고 데이터는 패션/의류 도메인에 맞춘다.
- 순환 재고, ESG, AI 리포트는 부가 기능이 아니라 핵심 기능으로 다룬다.
- 매장/창고/본사 흐름은 발주-입고-판매-이동-순환까지 연결된 라이프사이클로 본다.
- 임시 더미 데이터라도 향후 BE 연동을 고려한 구조와 의미를 유지한다.
