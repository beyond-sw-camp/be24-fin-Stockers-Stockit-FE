# 프로젝트 초기 분석 보고서

## 1. 프로젝트 개요

`StockIt ERP Frontend`는 재고, 발주, 물류, 매장 운영을 통합 관리하려는 멀티롤 ERP 프론트엔드다. 코드 구조상 사용자는 `본사(hq)`, `매장(store)`, `창고(warehouse)` 3개 역할로 나뉘며, 역할별로 다른 메뉴와 화면을 제공하는 것이 핵심이다.

이 프로젝트는 단순 쇼핑몰이 아니라, 본사 중심의 공급망 운영을 다루는 재고/발주/물류 운영 시스템에 가깝다. 여기에 `AI 리포트`, `ESG 현황판`, `순환재고` 같은 기능이 더해져 있어, 일반 ERP보다 운영 인텔리전스와 친환경 지표 시각화까지 확장하려는 방향성이 보인다.

## 2. 기술 스택 및 구조

프론트엔드는 `Vue 3 + Vite` 기반이며, 상태관리는 `Pinia`, 라우팅은 `Vue Router`, UI 스타일링은 `Tailwind CSS`, 차트는 `Chart.js + vue-chartjs`를 사용한다.

구조적으로는 다음 특징이 뚜렷하다.

- `src/router/index.js`: 역할 기반 라우팅과 접근 제어
- `src/config/roleMenus.js`: 역할별 메뉴 설정 중앙화
- `src/stores/*.js`: 도메인별 상태 관리
- `src/components/common/AppLayout.vue`: 공통 레이아웃
- `src/views/hq`, `src/views/store`, `src/views/warehouse`: 역할별 화면 분리

즉, 화면 단위로 흩어진 프로젝트가 아니라, 역할 기반 정보 구조와 도메인 스토어를 가진 관리자형 SPA로 설계되어 있다.

## 3. 주요 기능 분석

현재 구현 범위를 보면 아래 기능군이 확인된다.

- 본사 영역: 운영 대시보드, 재고 현황, 주문/발주 관리, 상품/거래처 관리, 계정 승인, AI 리포트, ESG 대시보드
- 매장 영역: POS 판매, 주문 관리, 매장 재고, 입고, AI 리포트
- 창고 영역: 창고 재고, 입고, 출고

특히 `StorePosView.vue`와 `inventory.js`를 보면 매장 판매 시 재고가 차감되고, 그 상태를 공유 스토어와 `localStorage`에 유지하는 흐름이 구현돼 있다. `purchaseOrder.js`, `hqAiReport.js`, `esg.js` 스토어도 실제 업무 화면처럼 구체적인 더미 데이터를 갖고 있어, 단순 와이어프레임이 아니라 데모 가능한 업무 시뮬레이션형 MVP에 가깝다.

## 4. 아키텍처 및 동작 방식 평가

이 프로젝트의 강점은 역할 기반 UX가 명확하다는 점이다. 로그인 후 역할별 홈으로 보내고, 허용되지 않은 경로 접근을 라우터 가드에서 차단한다. 공통 레이아웃과 메뉴 구성이 분리돼 있어 확장성도 나쁘지 않다.

또한 스토어 중심 구조 덕분에 다음 같은 흐름이 자연스럽다.

- 인증: `auth.js`
- 재고: `inventory.js`
- 발주: `purchaseOrder.js`
- AI 분석: `hqAiReport.js`
- ESG 지표: `esg.js`

즉, 현재 프로젝트는 "페이지 모음"보다 업무 도메인별 상태를 가진 ERP 프론트엔드 프로토타입으로 보는 것이 맞다.

## 5. 현재 완성도 판단

이 프로젝트는 운영 배포 직전 단계라기보다, 프론트엔드 MVP 또는 프로토타입 단계로 판단된다. 근거는 다음과 같다.

- 로그인은 실제 API가 아니라 더미 계정 기반
- 여러 API 파일이 실제 호출 대신 주석 스텁 상태
- 데이터 상당수가 스토어 내부 시드 또는 랜덤 생성 데이터
- 상태 저장이 서버가 아니라 `localStorage` 중심
- `README.md`는 기본 Vite 템플릿 수준으로 문서화가 부족

반면 CI/CD 흔적은 있다. `CICD/docker/Dockerfile`, `CICD/jenkins/jenkinsfile.frontend`, `CICD/k8s/*` 설정이 있어, 팀이 이 프로젝트를 실제 배포 가능한 형태로 발전시키려는 의도는 분명하다.

## 6. 눈에 띄는 특징과 리스크

가장 흥미로운 점은 일반 ERP 기능 외에 AI 운영 리포트와 ESG 대시보드가 포함되어 있다는 점이다. 이는 StockIt이 단순 재고 시스템이 아니라, 장기적으로는 운영 의사결정 지원 플랫폼을 지향할 가능성을 보여준다.

다만 다음 리스크도 보인다.

- 백엔드 연동이 아직 없어 실제 업무 시스템으로는 미완성
- 도메인 데이터가 일부 화면 간 완전히 일치하지 않음
- 재고/발주는 전자제품·문구 중심인데, AI/ESG는 의류·소재 중심 데이터가 섞여 있음
- `README.md`와 실행 문서가 부족해 온보딩 효율이 낮음

즉, 기능 아이디어는 풍부하지만, 데이터 모델과 실제 API 계약은 아직 정제 중인 상태로 보인다.

## 7. 종합 결론

이 프로젝트는 `StockIt`이라는 이름의 역할 기반 ERP 프론트엔드 MVP이며, 본사-매장-창고를 연결하는 재고/발주/물류 운영을 핵심으로 한다. 여기에 AI 추천과 ESG 시각화를 결합해, 단순 운영 툴이 아니라 데이터 기반 운영관리 플랫폼으로 확장하려는 기획 의도가 담겨 있다.

현재 기준으로는 "실서비스 완성본"보다는 데모 가능한 프론트 프로토타입에 가깝고, 다음 단계는 백엔드 연동, 데이터 모델 통일, 문서 정비가 될 가능성이 높다.

## 8. 검증 메모

`npm.cmd run build`로 확인했을 때 현재 환경에서는 Tailwind 네이티브 바이너리 로드 및 권한 관련 문제로 빌드 성공까지는 확인하지 못했다.

- `@tailwindcss/oxide-win32-x64-msvc` 로드 실패
- `spawn EPERM` 발생

이는 코드 자체 문제라기보다 현재 실행 환경 또는 권한 제약의 영향일 가능성이 크다.
