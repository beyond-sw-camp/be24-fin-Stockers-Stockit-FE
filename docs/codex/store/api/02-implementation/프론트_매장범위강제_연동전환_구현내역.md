# 프론트 매장 범위 강제 연동 전환 구현 내역

## 1. 작업 개요
- 백엔드 신계약(로그인 매장 컨텍스트 강제) 반영
- 판매/발주 화면에서 `storeCode/storeLocationId` 입력 의존 제거
- 매장 화면의 HQ API 의존 제거 및 Store API 기반으로 전환

## 2. 구현 범위
- 판매
  - `src/views/store/sales/StorePosView.vue`
  - `src/views/store/sales/StoreSalesHistoryView.vue`
- 발주
  - `src/views/store/orders/StoreOrderRequestView.vue`
  - `src/views/store/orders/StoreOrderHistoryView.vue`
  - `src/views/store/orders/StoreOrderAnalysisView.vue`

## 3. 주요 변경 사항
### 3.1 API 계약 정렬
- 판매 생성 요청에서 `storeCode` 제거
- 발주 생성/수정 요청에서 `storeCode`, `storeLocationId` 제거
- 판매/발주 목록 및 발주 분석 조회에서 `storeCode` query 제거

### 3.2 인증 사용자 기준 필드 통일
- 매장 화면의 사용자 매장 식별 기준을 `auth.user.locationCode/locationName`으로 통일
- 기존 `auth.user.storeCode/storeName/storeLocationId` 참조 제거

### 3.3 재고 조회 경로 전환
- `StorePosView`, `StoreOrderRequestView`에서 HQ SKU API 의존 제거
  - 제거: `@/api/hq/productMaster.js` (`getProductSkus`)
  - 사용: `@/api/store/inventory.js` (`getStoreInventories`, `getStoreInventorySkus`)
- SKU 목록 매핑 시 단가는 `sku.unitPrice` 기준으로 반영

## 4. 장애 원인 및 조치
### 4.1 증상
- 판매 등록 페이지 진입 시 GET 오류 다수 발생
- 인증 세션 이탈 후 로그인 화면으로 이동

### 4.2 원인
- 매장 권한 화면에서 `/api/hq/*` 경로를 호출
- 권한 차단(403/401 처리)으로 연쇄 실패 발생

### 4.3 조치
- 매장 화면 내 HQ API 호출 제거
- 매장 전용 Store API만 사용하도록 정리

## 5. 검증 결과
- 코드 검색 기준:
  - `src/views/store` 및 관련 store 연동 코드에서 `hq/productMaster`, `getProductSkus`, `/api/hq/` 호출 제거 확인
  - 판매/발주 API 요청에 `storeCode/storeLocationId` 전달 코드 미검출
- 참고:
  - 로컬 빌드(`vite build`)는 개발 PC 실행 환경 이슈(ExecutionPolicy/oxide 로딩)로 별도 환경 점검 필요

## 6. 후속 체크 포인트
- 매장 계정별 로그인 시 각 계정의 `locationCode` 범위 데이터만 노출되는지 QA
- 백엔드 전용 범위 위반 에러코드 수신 시 화면 메시지 정책(토스트/가드 문구) 최종 점검
