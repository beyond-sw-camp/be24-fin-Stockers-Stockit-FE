# 매장_입고영역_구현_내역

## 1. 구현 개요

매장 입고 기능을 기존 placeholder 화면에서 실제 사용 가능한 `입고 리스트 / 입고 상세 / 입고 분석` 구조로 확장했다.

이번 구현의 핵심 목표는 아래와 같다.

- 승인 완료된 발주를 입고 흐름으로 연결
- 입고 진행 상태와 완료 상태를 한 화면에서 조회 가능하게 구성
- 입고 확정 시 실제 매장 재고에 반영되도록 연결
- 발주 종료 상태를 `COMPLETED`로 정리해 입고 완료와 발주 완료를 일치시킴

본 문서는 기능 구현 기준을 정리한 문서이며, 배지 위치나 애니메이션 같은 후반 디자인 디테일 조정은 제외한다.

## 2. 메뉴 및 라우트 반영

### 2.1 메뉴 구조

매장 메뉴에 `입고 관리`를 추가하고 하위 메뉴를 아래처럼 구성했다.

- 입고 리스트
- 입고 분석

별도 `입고 내역` 메뉴는 두지 않고 입고 리스트 안에서 완료 건까지 함께 조회하도록 정리했다.

### 2.2 라우트 구조

아래 라우트를 반영했다.

- `/store/inbound` -> `/store/inbound/list`
- `/store/inbound/list`
- `/store/inbound/list/:id`
- `/store/inbound/history` -> `/store/inbound/list`
- `/store/inbound/history/:id` -> 입고 상세 리다이렉트
- `/store/inbound/analysis`

관련 파일:

- `src/router/index.js`

## 3. 스토어 및 상태 모델 구현

### 3.1 source of truth

입고 전용 별도 스토어를 새로 만들지 않고 `src/stores/storeOrders.js`를 source of truth로 유지했다.

발주 엔티티에 입고 상태와 입고 메타데이터를 확장하는 방식으로 구현했다.

### 3.2 상태 모델 정리

발주 상태:

- `REQUESTED`
- `APPROVED`
- `COMPLETED`
- `CANCELLED`

입고 상태:

- `READY_TO_SHIP`
- `IN_TRANSIT`
- `ARRIVED`
- `RECEIVED`

의미:

- `APPROVED`는 승인 완료 후 입고 진행 중
- `COMPLETED`는 입고 확정까지 끝난 종료 상태

### 3.3 입고 관련 필드

`StoreOrder`에 아래 필드를 반영했다.

- `inboundStatus`
- `inboundExpectedAt`
- `inboundCompletedAt`
- `inboundConfirmedBy`
- `inboundStatusHistory[]`

`StoreOrderItem`에는 아래 필드를 반영했다.

- `expectedInboundQuantity`

### 3.4 입고 관련 액션

아래 액션을 구현했다.

- `markReadyToShip(orderId)`
- `markInTransit(orderId)`
- `markArrived(orderId)`
- `confirmInbound(orderId, confirmedBy)`

`confirmInbound()` 동작:

- `ARRIVED` 상태에서만 허용
- `inventory.receiveItems()` 호출
- 발주 상태를 `COMPLETED`로 변경
- 입고 상태를 `RECEIVED`로 변경
- `inboundCompletedAt`, `inboundConfirmedBy` 저장
- `statusHistory[]`, `inboundStatusHistory[]` 갱신

### 3.5 시드 데이터 확장

입고 리스트와 분석 화면을 확인할 수 있도록 상태별 시드 데이터를 확장했다.

- `READY_TO_SHIP`
- `IN_TRANSIT`
- `ARRIVED`
- `RECEIVED`

또한 완료 건은 모두 `COMPLETED + RECEIVED` 조합으로 정리했다.

## 4. 입고 리스트 구현

관련 파일:

- `src/views/store/inbound/StoreInboundListView.vue`

반영 내용:

- 상태 탭: `전체 / 배송 준비중 / 배송 중 / 배송 완료 / 입고 완료`
- 검색, 기간 필터, 정렬
- 발주일시, 발주번호, 대표 상품명, 총 SKU, 입고 수량, 입고 상태, 입고 예정일 표시
- `입고 확정 처리` 컬럼 추가

동작 규칙:

- `ARRIVED + APPROVED` 건에는 `입고 확정하기` 액션 노출
- `READY_TO_SHIP`, `IN_TRANSIT`는 `대기`
- `RECEIVED`, `COMPLETED`는 `처리 완료`
- 액션 클릭 시 즉시 확정이 아니라 입고 상세로 이동
- 행 클릭으로도 상세 이동 가능
- 액션 버튼 클릭은 행 클릭과 분리 처리

추가 반영:

- 배송 완료 상태 건수가 있으면 상단에 `입고 확정 필요` 배너 노출
- `배송 완료만 보기`
- `첫 발주 바로 처리`

## 5. 입고 상세 구현

관련 파일:

- `src/views/store/inbound/StoreInboundDetailView.vue`

반영 내용:

- 발주번호, 발주일시, 매장, 입고 상태 배지 표시
- 기본 정보 카드
- SKU 라인 테이블
- 발주 메모
- 입고 정책 카드
- 입고 진행 이력 타임라인
- 입고 확정 버튼 및 확인 모달

세부 규칙:

- 상단 헤더 배지는 입고 상태만 노출
- `배송 완료` 상태에서만 입고 확정 버튼 활성화
- `배송 완료` 상태로 상세에 진입하면 상단 전체 폭 안내 배너 노출
- 안내 배너에는 아래 메시지를 포함
  - 실제 입고 수량과 발주 수량 확인 후 확정
  - 입고 확정 시 재고 즉시 반영

타임라인:

- 입고 상태 이력을 시간순으로 표시
- 현재 마지막 진행 단계에만 하이라이트 애니메이션을 적용
- 현재 상태가 `RECEIVED`일 때는 추가 하이라이트를 주지 않음

## 7. 발주/재고 연계 반영

### 7.1 발주 연계

`storeOrders.js`에서 발주 완료 의미를 `COMPLETED`로 확장했다.

반영 결과:

- 발주 내역에 `완료` 탭 추가
- 입고 확정 후 발주 내역의 `승인 완료` 탭에서 `완료` 탭으로 이동
- 발주 상세에서 `입고까지 완료된 종료 발주` 문구 노출
- 발주 분석에 `completedCount` 반영

관련 화면:

- `src/views/store/orders/StoreOrderHistoryView.vue`
- `src/views/store/orders/StoreOrderDetailView.vue`
- `src/views/store/orders/StoreOrderAnalysisView.vue`

### 7.2 재고 연계

`src/stores/inventory.js`에 `receiveItems(items)`를 사용해 입고 확정 시 SKU 재고를 증가시키도록 연결했다.

추가로 재고 조회 화면도 실제 `inventory` 스토어 기반으로 정리해 입고 반영 결과를 화면에서 이어서 확인할 수 있게 맞췄다.

관련 파일:

- `src/stores/inventory.js`
- `src/views/store/inventory/StoreInventoryView.vue`
- `src/views/store/inventory/StoreInventorySkuDetailView.vue`

## 10. 현재 구현 기준 정리

현재 매장 입고 기능은 아래 흐름으로 동작한다.

1. 승인 완료된 발주가 입고 리스트로 노출된다.
2. 입고 리스트에서 진행 상태와 완료 상태를 함께 조회한다.
3. 배송 완료 건은 상단 배너와 `입고 확정 처리` 컬럼으로 별도 안내된다.
4. 사용자는 입고 상세로 이동해 발주 라인과 입고 상태를 확인한다.
5. 배송 완료 상태에서는 안내 배너를 확인한 뒤 입고 확정을 진행한다.
6. 입고 확정 시 매장 재고가 증가하고, 발주 상태는 `COMPLETED`, 입고 상태는 `RECEIVED`로 변경된다.
7. 완료 건은 발주 내역과 입고 리스트 양쪽에서 종료 상태로 조회된다.

## 12. 창고 출고 연계 메모

- 매장 입고 상태 전환 전단계(출고 준비/배송중)는 `warehouseOutbound` 스토어와 창고 출고 화면에서 관리한다.
- 출고 상세 라우트: `/warehouse/outbound/:id`
- 출고 구현 문서: `docs/codex/warehouse/01-plan/물류창고_출고영역_통합_설계안.md`, `docs/codex/warehouse/02-implementation/물류창고_출고영역_구현_내역.md`

## 13. 매장 발주 연계 경계(출고 통합 기준)

- `STORE_OUTBOUND` 유형은 `storeOrders`를 source of truth로 사용한다.
- 창고 출고 화면은 `storeOrders.inboundStatus`를 읽어 `READY_TO_SHIP / IN_TRANSIT / COMPLETED`로 매핑한다.
- `WH_TRANSFER`, `CIRCULAR_SALE` 유형은 매장 발주 도메인 바깥 영역이며, 현재는 `warehouseOutbound`의 mock adapter로만 조회한다.
- 따라서 매장 발주/입고 로직은 `STORE_OUTBOUND` 경로에서만 상태 변경 액션과 직접 연결된다.
