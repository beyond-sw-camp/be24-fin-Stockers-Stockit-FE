# 매장_발주영역_구현_내역

## 1. 구현 개요

매장 발주 기능을 기존 단일 placeholder 화면에서 실제 동작 가능한 `발주 요청 / 발주 내역 / 발주 상세 / 발주 분석` 구조로 확장했다.

이번 구현의 핵심 목표는 아래 4가지였다.

- 매장 발주를 본사 거래처 발주와 분리된 별도 도메인으로 정리
- 매장 재고/판매와 동일한 의류 SKU 기준으로 요청 흐름 구현
- 요청, 조회, 수정, 취소, 분석의 기본 사용자 흐름 완성
- 이후 입고 관리와 연결 가능한 상태/이력/헤더-라인 구조 확보

## 2. 메뉴 및 라우트 반영

### 2.1 메뉴 구조

매장 `발주 관리` 하위 메뉴를 아래처럼 분리했다.

- 발주 요청
- 발주 내역
- 발주 분석

### 2.2 라우트 구조

아래 라우트를 추가/정리했다.

- `/store/orders` -> `/store/orders/request`
- `/store/orders/request`
- `/store/orders/request/:id/edit`
- `/store/orders/history`
- `/store/orders/history/:id`
- `/store/orders/analysis`

## 3. 스토어 및 데이터 모델 구현

### 3.1 매장 발주 전용 스토어 신설

파일:

- `src/stores/storeOrders.js`

반영 내용:

- 매장 발주 전용 Pinia 스토어 추가
- `StoreOrder + StoreOrderItem[]` 구조 도입
- localStorage 저장/로드 지원
- 시드 데이터 포함
- 발주번호 자동 생성
- 상태 이력 관리
- 목록/분석 getter 제공

### 3.2 상태 모델

내부 상태값:

- `REQUESTED`
- `APPROVED`
- `CANCELLED`

UI 노출 문구:

- `승인 대기`
- `승인 완료`
- `취소`

동작 규칙:

- 수정 가능: `REQUESTED`
- 취소 가능: `REQUESTED`
- `APPROVED`, `CANCELLED` 상태에서는 수정/취소 차단

### 3.3 재고 표시 필드

발주 라인에 아래 필드를 반영했다.

- `itemCode`
- `currentStoreStock`
- `inboundExpectedQuantity`
- `availableStoreStock`
- `safetyStock`
- `recommendedQuantity`

계산 규칙:

- `availableStoreStock = currentStoreStock + inboundExpectedQuantity`
- `recommendedQuantity = max(0, safetyStock - currentStoreStock)`

## 4. 발주 요청 화면 구현

파일:

- `src/views/store/orders/StoreOrderRequestView.vue`

### 4.1 화면 구조

발주 요청 화면을 좌우 2단 구조로 구현했다.

- 좌측: 매장 발주 검색
- 우측: 발주 요청서

### 4.2 검색/필터/정렬 기능

검색 및 필터:

- 상품명 검색
- 대분류 필터
- 소분류 필터
- 색상 필터
- 사이즈 필터

정렬:

- 부족 SKU 우선
- 카테고리순
- 상품명순
- 재고 부족한순
- 재고 많은순

대분류 고정 순서:

- 상의
- 바지
- 치마
- 아우터

이 순서는 아래 영역에 동일하게 반영했다.

- 대분류 필터 옵션
- 카테고리순 정렬
- 카테고리 기반 분석 집계 순서

### 4.3 SKU 테이블 컬럼

- 품목코드
- 상품명
- 옵션
- 카테고리
- 실재고
- 가용재고
- 안전 재고
- 권장 발주량
- 상태
- 추가

### 4.4 상태 표시

SKU 상태값은 화면에서 아래 3개만 사용한다.

- 부족
- 정상
- 품절

### 4.5 요청서 동작

반영 내용:

- 같은 SKU 재추가 시 요청 수량 누적
- 품목코드 표시
- 품목코드가 상품명보다 먼저 보이도록 정보 순서 조정
- 실재고 / 가용재고 / 안전재고 / 권장 수량 노출
- 요청 수량 직접 입력 및 증감 버튼
- 총 SKU / 총 요청 수량 요약
- 전체 비우기
- 메모 입력
- 발주 요청 등록
- 수정 모드 재사용

운영 정책 영역:

- 등록 버튼과 정책 안내 순서 정리
- 버튼/정책 안내 간 여백 및 구분선 보강
- 자동 승인 기능은 구현하지 않고 안내 문구만 노출

### 4.6 UI 조정 사항

반영한 주요 세부 조정:

- 검색 패널 내 컬럼 간격 축소
- 가로 스크롤이 생기지 않도록 테이블 폭 재조정
- `담기` 버튼 폭 축소
- `담기` 버튼 글자색을 흰색으로 고정
- 요청서 패널 폭 소폭 축소
- 총 SKU / 총 요청 수량 요약 간격 조정

## 6. 발주 상세 화면 구현

파일:

- `src/views/store/orders/StoreOrderDetailView.vue`

### 6.1 상세 구조

상세 페이지를 별도 라우트로 추가했다.

- 좌측: 발주 기본정보, SKU 라인, 메모, 취소 사유
- 우측: 정책 안내, 진행 이력, 수정/취소 영역

### 6.2 상세 SKU 테이블

반영 컬럼:

- 품목코드
- 상품명
- 옵션
- 카테고리
- 실재고
- 가용재고
- 요청 수량

조정 내용:

- 기존 상품명 아래에 붙어 있던 카테고리를 독립 컬럼으로 분리
- 품목코드 추가
- 컬럼 간격 축소

### 6.3 우측 패널

반영 내용:

- 정책 안내
- 진행 이력
- 수정/취소 버튼
- 처리 결과 메시지

레이아웃 조정:

- 상세 바깥 2단 비율을 `70 : 30`으로 조정
- 오른쪽 칼럼 내부 카드도 모두 칼럼 폭을 꽉 채우도록 정리

### 6.4 액션 규칙

- `REQUESTED` 상태에서만 수정/취소 버튼 노출
- 수정 버튼은 요청 수정 화면으로 이동
- 취소는 상세 페이지 내 모달로 처리

## 11. 현재 구현 기준 정리

현재 매장 발주 기능은 아래 흐름까지 동작한다.

1. 매장 발주 검색에서 SKU를 검색/필터/정렬한다.
2. SKU를 요청서에 담고 수량/메모를 조정한다.
3. 발주 요청을 등록한다.
4. 발주 내역 목록에서 발주건을 조회한다.
5. 발주 상세에서 진행 이력과 SKU 라인을 확인한다.
6. 승인 대기 상태라면 상세에서 수정/취소를 수행한다.
7. 발주 분석에서 누적 현황을 확인한다.
